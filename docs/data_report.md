## Interpretation: implications for the loss

The class distribution is heavily imbalanced: the sea-surface background dominates while the headline `Oil Spill` class occupies a small fraction of pixels. A plain (unweighted) cross-entropy loss would be minimised by predicting mostly background and would under-segment oil, so it is not a good fit here.

These numbers motivate an imbalance-aware loss for the segmentation stage: class-weighted cross-entropy (e.g. inverse-frequency or effective-number weights derived from the table above), and/or a region-overlap loss such as Dice combined with Focal loss to focus learning on the rare oil and look-alike classes. The exact weights should be derived from the train pixel counts reported here. Selection and reporting are on oil-class IoU and recall (see `oilspill.metrics`), not pixel accuracy, precisely because of this imbalance.

## Auxiliary classification dataset: usefulness decision

**Decision: do NOT use the auxiliary classification dataset (`data/raw/oil_spill_classification_dataset.zip`) for training or evaluation in this project.**

Rationale. Inspecting the archive shows it is not an independent dataset: it is the *same* MKLab oil-spill data re-organised into binary oil / no-oil folders. Its `train` split contains 791 `Images_Oil` + 211 `Images_No_Oil` = 1002 images and its `test` split 88 + 22 = 110 images, exactly matching the segmentation train (1002) and test (110) counts, with matching `img_XXXX` filename stems and identical `Labels_1D`/`Labels` folders. It therefore adds no new imagery.

Consequences for the candidate uses:

- **Auxiliary classification pretraining of the encoder** would pretrain on the very images later used for segmentation training, with no held-out benefit and a real risk of leaking test images into pretraining (the classification `train` includes images that map to the segmentation set). Not worthwhile.
- **Extra evaluation set**: it is the same images and the same official split, so it provides no independent evaluation signal.
- **Hard-negative / look-alike mining**: the binary oil/no-oil label is coarser than the existing 5-class masks (which already separate Oil from Look-alike). The segmentation masks we already load are strictly more informative, so the binary split offers nothing extra here either.

If a future need arises for an image-level oil/no-oil signal, it can be derived directly from the segmentation masks already loaded (oil present iff the oil class appears), avoiding this redundant, leakage-prone archive.
